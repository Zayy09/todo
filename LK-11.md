# LK-11: Laporan Bug Fix & Refactoring

## 1. Identifikasi Bug / Masalah
1. **Dead Code pada AuthController:** Terdapat method `login` yang menangani login tradisional (email dan password) padahal aplikasi sudah menggunakan WorkOS SSO. Hal ini berpotensi membingungkan developer lain dan merupakan *technical debt*.
2. **Information Disclosure pada Routes:** Terdapat route debug (`/cek-redirect` dan `/cek-workos`) di `routes/web.php` yang secara langsung menampilkan nilai dari *environment variable*. Ini adalah celah keamanan minor yang dapat mengekspos konfigurasi ke publik.
3. **Penggunaan `env()` di Controller:** Pemanggilan fungsi `env()` secara langsung di dalam `WorkOSController`. Pada Laravel, penggunaan `env()` di luar file konfigurasi (folder `config/`) sangat tidak disarankan. Jika *config cache* diaktifkan (`php artisan config:cache`), pemanggilan `env()` akan mengembalikan nilai `null` dan menyebabkan fitur SSO *error*.

## 2. Dokumentasi Perbaikan (Sebelum - Sesudah)

### A. Dead Code AuthController
- **Sebelum:** Terdapat method `login(Request $request)` di `AuthController` dan route `Route::post('/login')` di `routes/web.php` yang tidak digunakan karena otentikasi sudah digantikan sepenuhnya oleh halaman SSO WorkOS.
- **Sesudah:** Menghapus method `login` dari `AuthController` dan menghapus `Route::post('/login')` di `routes/web.php` untuk menjaga kebersihan *codebase*.

### B. Information Disclosure Routes
- **Sebelum:** Terdapat route `/cek-redirect` dan `/cek-workos` yang memanggil fungsi `dd(env(...))` dan dapat diakses publik.
- **Sesudah:** Menghapus kedua route tersebut dari `routes/web.php` agar informasi rahasia tidak terekspos.

### C. Penggunaan `env()` di Controller
- **Sebelum:** `WorkOSController` memanggil `env('WORKOS_API_KEY')`, `env('WORKOS_CLIENT_ID')`, dan `env('WORKOS_REDIRECT_URL')` secara eksplisit pada inisialisasi *client*.
- **Sesudah:** Menambahkan array konfigurasi `workos` di dalam `config/services.php` dan memperbarui `WorkOSController` untuk menggunakan fungsi pembantu bawaan `config('services.workos.api_key')`.

## 3. Refactoring Code

1. **Pembuatan Form Request Validation (TodoRequest):**
   - **Masalah:** Terdapat duplikasi logika validasi yang cukup tebal dan identik di dalam method `store` dan `update` pada `TodoController`.
   - **Refactoring:** Membuat custom request class `TodoRequest` (`php artisan make:request TodoRequest`) yang memusatkan aturan validasi (`rules`) dan pesan *error* (`messages`). Method `store` dan `update` di controller kini menjadi jauh lebih ramping dan bersih berkat *type-hint* `TodoRequest $request`.
   
2. **Standarisasi Konfigurasi Eksternal (WorkOS):**
   - **Masalah:** Inisialisasi kredensial `WorkOS` di-hardcode menggunakan fungsi bawaan global tanpa sentralisasi konfigurasi.
   - **Refactoring:** Mengekstraksi kredensial ke dalam file konfigurasi Laravel standar (`config/services.php`). Selain memperbaiki masalah *caching*, ini memisahkan struktur konfigurasi dari logika *controller* dan menjadikan *codebase* lebih terstruktur.
